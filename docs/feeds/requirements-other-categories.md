# Требования к фиду. Другие категории

Фид «Отели и аренда жилья» Google Рекламы

Фид «Отели и аренда жилья» Google Рекламы должен быть в формате CSV. Первая строка содержит названия столбцов, а следующие строки ― сами данные. Содержимое столбцов отделяется друг от друга запятой. Данные в фиде должны быть в кодировке UTF-8.

**Пример:**

[Скачать пример фида для отелей в формате CSV](https://yastatic.net/s3/doc-binary/src/support/direct/ru/hotels_feed_example.csv)

**Элементы, входящие в CSV**

В таблице ниже описаны элементы фида, которые используются для создания товарных объявлений и объявлений для страниц каталога в Единой перфоманс-кампании.

|  |  |
| --- | --- |
| **Элемент** | **Описание** |
| `Property ID` | Идентификатор отеля. Обязательный элемент. |
| `Category` | Тип размещения. Например:   - «отель» - «апартаменты» - «хостел» |
| `Property name` | Название отеля. Обязательный элемент. |
| `Final URL` | URL страницы предложения для продвижения сайта или трекинговая ссылка для продвижения мобильного приложения. Если вы продвигаете мобильное предложение, рекомендуем добавить отложенные диплинки. Обязательный элемент. |
| `Destination name` | Местоположение отеля (не более 25 символов). Обязательный элемент. |
| `Image URL` | Ссылка на изображение. Рекомендуемый размер изображения — минимум 500 пикселей по одной из сторон. Обязательный элемент для показов в РСЯ. |
| `Price` | Цена предложения. Число и код валюты (`RUB`, `USD`, `UAH`, `KZT`). Используйте точку (.) в качестве десятичного разделителя. Рекомендуемый элемент. |
| `Star rating` | Количество звезд. Целое число от 1 до 5. |
| `Score` | Пользовательская оценка, число. Используйте точку (.) в качестве десятичного разделителя. |
| `Max score` | Максимально возможная оценка, целое число. По умолчанию 5. |
| `Facilities` | Услуги отеля. Укажите значения через точку с запятой: `бар;бассейн;бесплатный Wi-Fi`. |
| - `custom_label_0` - `custom_label_1` - `custom_label_2` - `custom_label_3` - `custom_label_4` | Кастомные элементы. Произвольное описание. Латинские и кириллические буквы, цифры. Длина одного элемента — до 175 символов.  Кастомные элементы фида могут использоваться только для создания фильтров в Единой перфоманс-кампании. На генерацию креативов они не влияют. Фильтры используются для сегментирования оферов по специфичным параметрам, для которых не предусмотрены стандартные фильтры. Например, можно задать фильтр по принадлежности отеля к конкретной акции. |
| `custom_score` | Кастомный элемент. Цифровое значение — ноль или целое положительное число.  Кастомные элементы фида могут использоваться только для создания фильтров в Единой перфоманс-кампании. На генерацию креативов они не влияют. Фильтры используются для сегментирования оферов по специфичным параметрам, для которых не предусмотрены стандартные фильтры. Например, можно задать фильтр по маржинальности предложения. |

Примечание

Значения, содержащие запятую, нужно заключить в кавычки (""). Например, "Москва, центр".

Фид Авто.ру

Фид Авто.ру должен быть в формате XML. Данные в фиде должны быть в кодировке UTF-8.

В фиде можно передать информацию о каталогах автомобилей. В общей структуре файла элемент `<collections>` идет после элемента `<cars>`.

**Элементы, входящие в** `<collections>`

|  |  |  |
| --- | --- | --- |
| **Элемент** | **Описание** | **Обязательный элемент** |
| `collection id` | Уникальный ID каталога, не более 60 символов. Может быть числовым, буквенным, буквенно-числовым.  Идентификаторы для каждого каталога должны быть уникальны. | да |
| `url` | URL страницы каталога.  Максимальная длина ссылки — 2048 символов. Допускаются кириллические ссылки. (Кириллические ссылки рекомендуем преобразовать с помощью [Punycode](https://www.punycoder.com/).) URL‑адрес формируется на основе стандарта [RFC 3986](https://datatracker.ietf.org/doc/html/rfc3986).  В URL обязательно указать протокол `http` или `https`. Как кодировать символы в URL?  |  |  | | --- | --- | | **Символ в тексте** | **Код для YML-файла** | | `"` | `&quot;` | | `&` | `&amp;` | | `>` | `&gt;` | | `<` | `&lt;` | | `'` | `&apos;` |   URL не должен содержать пробелов. Их нужно преобразовать в `%20`. | да |
| `picture` | Ссылка на изображение.  Можно добавить несколько ссылок на изображения, каждую в собственном элементе picture. Как изменить картинку в объявлении? Чтобы робот Яндекса обновил изображение каталога в объявлениях, нужно изменить ссылку на него в фиде​. Например:   - ссылка на текущее изображение — `<picture>http://auto.ru/newavto/00016.jpg</picture>` - ссылка на новое изображение — `<picture>http://auto.ru/newavto/00016-new.jpg</picture>` | для показов в РСЯ |
| `name` | Название каталога. | да |
| `description` | Описание каталога. | нет |

**Пример для страниц каталога**

```
<?xml version="1.0" encoding="utf-8"?>
<data>
  <cars>
    <car></car>
    <car></car>
  </cars>
  <collections>
    <collection id="1">
      <url>https://auto.ru/new</url>
      <name>Новые автомобили в наличии</name>
      <description>Купить новые автомобили у официального дилера</description>
      <picture>http://auto.ru/newavto/00016.jpg</picture>
    </collection>
  </collections>
</data>
```

Фид Яндекс Недвижимости

Фид Яндекс Недвижимости должен быть в формате YRL (Yandex Realty Language). Подробнее о формате YRL в [Справке Яндекс Недвижимости](https://yandex.ru/support/realty/partners/requirements.html).

Каждое товарное предложение описывается отдельным элементом `<offer>`.

В одном фиде можно передавать информацию о разных ЖК разных застройщиков. Для этого укажите в каждом `<offer>` правовую информацию о застройщике в элементе `<sales-agent>`.

Например:   - ссылка на текущее изображение — `<picture>http://www.developer.ru/images/plans/000001279.jpg</picture>` - ссылка на новое изображение — `<picture>http://www.developer.ru/images/plans/000001279-new.jpg</picture>` | для показов в РСЯ |
| `name` | Название каталога. | да |
| `description` | Описание каталога. | нет |

**Пример для страниц каталога**

```
<?xml version="1.0" encoding="utf-8"?>
<realty-feed>
  <offer>
  </offer>
  <collections>
    <collection id="1">
      <url>https://developer.ru/#/macrocatalog/complexes/1111111</url>
      <name>Двухкомнатные квартиры на Васильевском острове</name>
      <description>Двухкомнатные квартиры на Васильевском острове с отделкой от застройщика и современной планировкой</description>
      <picture>http://www.developer.ru/images/plans/000001279.jpg</picture>
    </collection>
  </collections>
</realty-feed>
```

При создании объявлений максимально подробно наполняйте офферы всеми доступными элементами (тегами), даже если они не обязательны. Это поможет создать наиболее привлекательное объявление, которое будет отображаться во всех возможных форматах.

Длина одного элемента — до 175 символов.  Кастомные элементы фида могут использоваться только для создания фильтров в Единой перфоманс-кампании. На генерацию креативов они не влияют. Фильтры используются для сегментирования оферов по специфичным параметрам, для которых не предусмотрены стандартные фильтры. Например, можно задать фильтр по принадлежности предложения к конкретной акции. |
