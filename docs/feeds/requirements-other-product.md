# Требования к фиду. Продажа товаров. Прочие типы

Общие требования для товарных фидов:

- Каждый товар должен иметь уникальный идентификатор.
- Идентификаторы одного товара в разных фидах должны быть одинаковы, в том числе в и в фидах разных типов. Например, один и тот же товар в [фиде Яндекс Маркета](../feeds/requirements-yml.md) и в фиде Google Shopping должен иметь один и тот же идентификатор.

Фид в формате Google Shopping

Универсальный фид

Универсальный фид должен быть в формате CSV. Первая строка содержит названия столбцов, а следующие строки — сами данные. Содержимое столбцов отделяется друг от друга запятой. Данные в фиде должны быть в кодировке UTF-8.

**Пример**

[Скачать пример универсального фида в формате CSV](https://yastatic.net/s3/doc-binary/src/support/direct/ru/yandex_custom_feed_example.csv)

| - `custom_label_0` - `custom_label_1` - `custom_label_2` - `custom_label_3` - `custom_label_4` | Кастомные элементы. Произвольное описание. Латинские и кириллические буквы, цифры. Фильтры используются для сегментирования оферов по специфичным параметрам, для которых не предусмотрены стандартные фильтры. Например, для e-com можно задать фильтр по принадлежности товара к конкретной акции. |
| `custom_score` | Кастомный элемент. Цифровое значение — ноль или целое положительное число.  Кастомные элементы фида могут использоваться только для создания фильтров в Единой перфоманс-кампании. На генерацию креативов они не влияют. Фильтры используются для сегментирования оферов по специфичным параметрам, для которых не предусмотрены стандартные фильтры. Например, для e-com можно задать фильтр по маржинальности предложения. |

Фид «Специальный» Google Рекламы

Фид «Специальный» Google Рекламы должен быть в формате CSV. Первая строка содержит названия столбцов, а следующие строки — сами данные. Содержимое столбцов отделяется друг от друга запятой. Данные в фиде должны быть в кодировке UTF-8.
