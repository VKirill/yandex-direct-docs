## Получить все предложения банков

Получает актуальные предложения от банков-партнеров на основании номера телефона клиента. Является предпочтительным унифицированным каналом взаимодействия с источниками данных.

## Формат запроса

```
POST /integrations/v1/api/loan_approval
```

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| **Параметр** | **Описание** | **Тип** | **Обязательно** | **Комментарий** |
| `phone_number` | Номер телефона. | `string` | Да |  |
| `amount` | Сумма кредита. | `number` | Нет | Если продукт `"credit_card"`, то речь идет о кредитном лимите. |
| `term` | Срок кредита в месяцах. | `string` | Нет |  |
| `product_type` | Наименование кредитного продукта. | `string` | Да | Значение из справочника:   - `"loan"` — кредит наличными; - `"credit_card"` — кредитная карта. |
| `search_id` | `ID` запроса. | `string` | Да | Параметр для повторных запросов, `UUID`. |
| `props` | Дополнительные служебные параметры. | `array` | Нет | Массив строк, который может содержать любую служебную информацию, оговоренную вне данной спецификации. |

В случае асинхронной модели обработки данных:

В случае асинхронной модели обработки данных предполагается множественный опрос по `GET`. Выполняется, пока `status` в ответе не станет `done`.

```
GET /api/banking/loan_approval?search_id={{search_id}}
```

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| **Параметр** | **Описание** | **Тип** | **Обязательно** | **Комментарий** |
| `search_id` | `ID` запроса. | `string` | Да | Параметр для повторных запросов, `UUID`. |
