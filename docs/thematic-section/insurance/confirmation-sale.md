# Подтверждение продажи

- [Формат запроса](ru/thematic-section/insurance/confirmation-sale#request-format)
- [Успешный ответ](ru/thematic-section/insurance/confirmation-sale#200-ok)

Метод подтверждения факта продажи полиса.

Метод реализован на стороне Яндекса, токен доступа будет выдан единоразово на этапе подключения.

Важно

Вызов метода необходимо производить после продажи полиса для учета конверсии, но не позднее 11 дней после фактической оплаты. Рекомендуем передавать данные в течение 3 дней.

Может быть как GET, так и POST. Для POST доступна массовая отправка данных.

## Формат запроса

```
GET /confirm
```

|  |  |  |  |
| --- | --- | --- | --- |
| **Параметр** | **Описание** | **Тип** | **Комментарий** |
| `amount` | Сумма общей страховой премии. | `number` |  |
| `counterparty_id` | Идентификатор партнера, который выдал полис. | `integer` |  |
| `offer_id` | Идентификатор предложения. | `string` |  |
| `partner_id` | Идентификатор партнера, с которым настроена интеграция. | `string` |  |
| `request_id` | Идентификатор запроса. | `integer` |  |
| `is_upsaled` | Признак наличия кросс-продаж. | `boolean` |  |

## Успешный ответ

**200 OK**

Успешное подтверждение.

Формат ответа

```
{
    "request_id": "c3073b9d-edd0-49f2-a28d-b7ded8ff9a8b"
}
```

|  |  |  |  |
| --- | --- | --- | --- |
| **Параметр** | **Описание** | **Тип** | **Комментарий** |
| `request_id` | Идентификатор запроса. | `string` | `UUID`. |

#### Остались вопросы?

Внимание

Специалисты отдела клиентского сервиса могут вас проконсультировать только по кампаниям того логина, с которого вы обращаетесь. Логин можно увидеть, если открыть [ya.ru](http://ya.ru/) на соседней вкладке браузера. Специалист получит доступ к вашим данным только при обработке обращения.

Сканируйте QR-код или нажмите на него для перехода по ссылке.


При выборе Telegram, WhatsApp учитывайте, что Яндекс не контролирует, как сторонние мессенджеры хранят ваши данные и переписку на своей стороне, и не несет за это ответственность.
