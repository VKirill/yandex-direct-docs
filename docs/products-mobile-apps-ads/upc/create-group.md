# Шаг 2. Создание группы

Выберите аудиторию — настройте условия показа для этой группы объявлений:

1. Укажите географию показов — объявления будут показаны только той аудитории, которая сможет воспользоваться вашим предложением, так как находится в выбранном регионе показа или интересуется им.

   - Перечислить страны, области, города — в свободной форме начните вводить нужный регион, например, Тула или Нижний Новгород. Или воспользуйтесь деревом регионов.
   - Указать регион на карте — если хотите показывать объявления в конкретном месте, например, рядом с вашим кафе, либо вас интересуют клиенты в узкой области.

   [Подробнее о настройке](../../efficiency/geotargeting.md)
2. Настройте условия показа в соответствии с выбранной целью кампании.

   Привлечение новых пользователей

   Ретаргетинг

   - [Автотаргетинг](../../impression-criteria/autotargeting.md#at-for-search) — обязательное условие показа на поиске. В каждой группе объявлений нужно выбрать хотя бы одну категорию запросов. При переключении места показа с РСЯ на Поиск автотаргетинг будет включен автоматически в каждой группе на все категории запросов. Лучше при смене мест показа проверять его настройки. Отключить автотаргетинг можно только для кампаний с показами в сетях.
   - Чтобы ваши объявления показывались на поиске по запросам пользователей, задайте дополнительно ключевые фразы.

     При показе в сетях система автоматически выбирает объявления, ключевые фразы которых совпадают с тематикой площадки или интересами пользователя.
   - Для объявлений с показами в Рекламной сети Яндекса опишите [интересы и привычки](../../products-text-image-ads/user-profile.md#как-настроить-how-to) вашей аудитории: какими темами интересуется, какие сайты мог бы посещать и какие приложения мог бы использовать.
   - Добавьте условие формирования сегмента аудитории — в блоке **Сегменты аудитории** настройте комбинацию целей мобильного приложения и Метрики, а также сегментов Яндекс Метрики и сегментов Яндекс Аудиторий, с помощью которых можно описать нужную аудиторию. Для создания условий можно использовать готовые или автоматические сегменты, а также настроить работу с этими сервисами.

     - Сегменты аудитории на поиске работают совместно с ключевыми фразами или автотаргетингом через логический оператор «И». То есть объявления показываются пользователям, которые соответствуют и условиям формирования сегмента аудитории, и условиям показа: ключевым фразам или критериям автотаргетинга. Внутри условий сегменты аудитории работают через «ИЛИ». Если в условиях показа заданы ключевая фраза и 2 условия формирования сегмента аудитории, то объявление будет показано пользователю, который и указал в поиске фразу и попадает в сегмент аудитории.
     - Сегменты аудитории в сетях работают с другими условиями показа через оператор «ИЛИ». То есть объявление будет показываться пользователю, который или подходит хотя бы под одно из добавленных условий показа, или соответствует условиям формирования сегмента аудитории.

   **Место показа: Продвижение в поисковой выдаче**

   - [Автотаргетинг](../../impression-criteria/autotargeting.md#at-for-search) — обязательное условие показа на поиске. В каждой группе объявлений нужно выбрать хотя бы одну категорию запросов.
   - Чтобы ваши объявления показывались на поиске по запросам пользователей, задайте дополнительно ключевые фразы.
   - Добавьте условие ретаргетинга.

     ##### Способы создания ретаргетингового сегмента:

     - Динамический — данные о пользователях обновляются автоматически:

       - [На основе целей мобильного приложения](../../impression-criteria/retargeting-lists.md) – сегмент создается в Директе.
       - [Интеграция с AppsFlyer Audience](https://yandex.ru/support2/audience/ru/appsflyer) — сегмент формируется в Яндекс Аудиториях.
     - Статический — данные о пользователях обновляются только вручную:

       - [Cегмент на основе ID мобильных устройств](https://yandex.ru/support2/audience/ru/segments/create-from-file#id) — сегмент формируется в Яндекс Аудиториях.

       Примечание

       При создании сегмента в Яндекс Аудиториях на основе [ID мобильных устройств](https://yandex.ru/support2/audience/ru/segments/create-from-file#id) рекомендуем использовать режим **In-Device**.
   - Ретаргетинг на поиске работает совместно с ключевыми фразами или автотаргетингом через логический оператор «И». То есть объявления показываются пользователям, которые соответствуют и условиям подбора аудитории, и условиям показа: ключевым фразам или критериям автотаргетинга. Внутри условий подбор аудитории работает через «ИЛИ». Если в условиях показа заданы ключевая фраза и 2 условия подбора аудитории, то объявление будет показано пользователю, который и указал в поиске фразу, и подходит под одно из условий подбора аудитории.

   **Место показа: Рекламная сеть Яндекса**

   - Добавьте условие ретаргетинга. [Способы создания ретаргетингового сегмента](../../products-mobile-apps-ads/upc/create-group.md#create-segments) такие же, как для продвижения на поиске.
   - В указанном сценарии объявления в сетях показываются только тем пользователям, которые попадают под условие ретаргетинга.
3. В дополнительных настройках можно задать:

   - **Корректировки**

     С помощью корректировок вы можете повысить или понизить ставку при показе объявлений определенной аудитории. Корректировки позволяют получить больший объем трафика при показе выбранной аудитории, чем это произошло бы по изначально заданной ставке.

     [Как работают и какие могут быть установлены корректировки](../../products-mobile-apps-ads/bids-adjustment.md)
   - **Тип контента площадок**.

     Получите дополнительный охват за счет площадок Рекламной сети Яндекса, содержащих контент для взрослых. Тематика объявлений определяется на модерации.
