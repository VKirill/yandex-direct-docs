Получить данные о новых установках из магазина приложений можно с помощью трекинговой системы. Она сопоставляет установки и действия в приложении с переходами по объявлению и передает эти данные в Директ для подсчета *конверсии*.

Подключение трекинговой системы дает возможность:

- [оптимизировать](../strategies/average-cpa-mobile-apps.md) кампанию на установки и события в приложении;
- точно собирать статистику по установкам и событиям внутри приложения, которая будет доступна и в Директе, и в трекинговой системе.

Директ поддерживает следующие трекинговые системы: [AppMetrica](https://appmetrika.yandex.ru), [AppsFlyer](http://www.appsflyer.com/), [Adjust](https://www.adjust.com/), [Branch](https://branch.io/), [Kochava](http://www.kochava.com/), [MyTracker](https://tracker.my.com/), [Singular](https://www.singular.net/), [Tenjin](https://www.tenjin.com/) и [AirBridge](https://www.airbridge.io/). Поддержка SKAdNetwork для кампании на iOS 14.5 работает в системах AppMetrica, AppsFlyer и Adjust.

## Верификация приложения в трекинговой системе

Пройдите верификацию приложения, чтобы:

- настраивать ретаргетинг на пользователей, которые достигли целей, указанных в трекере;
- получать статистику по технологии SKAdNetwork для кампаний на iOS 14.5 и выше.

##### **AppMetrica**

Если AppMetrica и Директ находятся на разных логинах, для верификации приложения в кабинете AppMetrica необходимо выдать доступ логину Директа. Если логины совпадают, верификация будет пройдена автоматически.

##### **Adjust или AppsFlyer**

Укажите [App ID](../products-mobile-apps-ads/tracking-systems.md#добавьте-приложения-в-библиотеку-директа-add-aps) при настройке интеграции, и приложение будет верифицировано после получения первого постбека от мобильного трекера (вы увидите значок  в настройках приложения).

В случае с AppsFlyer, чтобы верификация прошла успешно и Директ получал обогащенные данные о конверсиях, в настройках интеграции в трекере с Директом отключите [фреймворк агрегированной расширенной конфиденциальности](https://support.appsflyer.com/hc/ru/articles/360018515798-%D0%A4%D1%80%D0%B5%D0%B9%D0%BC%D0%B2%D0%BE%D1%80%D0%BA-%D0%90%D0%B3%D1%80%D0%B5%D0%B3%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%BD%D0%BE%D0%B9-%D1%80%D0%B0%D1%81%D1%88%D0%B8%D1%80%D0%B5%D0%BD%D0%BD%D0%BE%D0%B9-%D0%BA%D0%BE%D0%BD%D1%84%D0%B8%D0%B4%D0%B5%D0%BD%D1%86%D0%B8%D0%B0%D0%BB%D1%8C%D0%BD%D0%BE%D1%81%D1%82%D0%B8).

## Добавьте приложение в библиотеку Директа

Чтобы настроить связь Директа и трекинговой системы, добавьте приложение в библиотеку.

1. В меню слева нажмите **Библиотека** и выберите **Мобильные приложения**. Нажмите **Добавить приложение**.
2. Укажите ссылку на приложение в AppStore или Google Play. Директ загрузит из магазина иконку, цену, рейтинг, количество оценок и технические характеристики приложения. Используйте App ID, полученный после добавления приложения, для настроек [трекинговых ссылок](../products-mobile-apps-ads/tracking-systems.md#tracking-links).

Обратите внимание, система получает данные для той страны, которая указана в ссылке на магазин приложений. По умолчанию данные загружаются для России. Используйте специальные параметры, чтобы объявление вело на магазин приложений для определенной страны.

AppStore

Google Play

Страна указывается в ссылке после `apps.apple.com`.

```
https://apps.apple.com/ru/app/yandex-browser-for-iphone/id483693909
```

Страна указывается в параметре `gl`.

```
https://play.google.com/store/apps/details?id=com.yandex.browser&gl=ru
```

После настройки трекинговой ссылки, вернитесь в библиотеку приложений и укажите ее для нужного приложения. Создавать кампании для приложения можно только с трекинговыми ссылками, которые указаны в библиотеке приложений. Если вы хотите использовать разные трекинговые системы для одного приложения, например, вы работаете с приложением и в [AppMetrica](https://appmetrika.yandex.ru), и в [Appsflyer](http://www.appsflyer.com/), указать их вместе для одного приложения не получится. Поэтому добавьте приложение в библиотеку 2 раза и укажите в одном случае трекинговую ссылку из одного трекера, а в другом — из другого.

## Трекинговые ссылки

### Продвижение на мобильных устройствах и планшетах

Чтобы связать трекинговую систему и кампании в Директе, необходимо добавить трекинговую ссылку в объявления. При этом в коде мобильного приложения должен быть прописан SDK трекинговой системы: это программный модуль, который передает системе информацию об установках и запусках.

В ссылке можно передавать различные параметры и метки, в том числе UTM-метки:

```
https://appmetrica.yandex.com/serve/7438548069196542965/?click_id={logid}&source={source}
```

С их помощью можно отслеживать различные статические показатели, например, источники трафика. Подробнее об [использовании параметров и меток](../statistics/url-tags.md).

Важно

Ссылка создается рекламодателем в трекере.

На вкладках ниже описано, как создать трекинговые ссылки в поддерживаемых Директом трекинговых системах.

Внимание

Если вы используете другую трекинговую систему, вы можете обратиться к ее владельцу (разработчику) с просьбой об интеграции с Директом. Заявки на подключение трекинговых систем к Директу принимаются только от владельцев этих систем через [форму обратной связи](../troubleshooting/tracking-system.md). Взаимодействие Директа с трекинговыми системами подробно описано в [документации](https://yastatic.net/s3/doc-binary/src/support/direct/ru/yandex-mobileapp-postback-guide.pdf).

Чтобы получить трекинговую ссылку, выполните следующие шаги:

1. Перейдите на вкладку **Трекеры** и нажмите **+Создать трекер**.
2. Выберите приложение в выпадающем списке **Приложения**. SDК уже должен быть прописан в коде приложения. Подробнее о том, как добавить SDK для [Android](https://appmetrica.yandex.ru/docs/mobile-sdk-dg/android/about/android-initialize.html) и [iOS](https://appmetrica.yandex.ru/docs/mobile-sdk-dg/ios/ios-quickstart.html).
3. Выберите Yandex.Direct в выпадающем списке **Партнер**.
4. Вставьте полученную трекинговую ссылку в объявление.

Должна получиться ссылка вида:

```
https://appmetrica.yandex.com/serve/7438548069196542965/?click_id={logid}
```

Ссылка должна обязательно содержать параметр `click_id={logid}`.

Подробнее см. в разделе [Создание трекера](https://tech.yandex.ru/appmetrica/doc/mobile-tracking/concepts/add-tracker-docpage/) документации [AppMetrica](https://tech.yandex.ru/appmetrica/).

Для корректной работы стратегии [Максимум конверсий](../strategies/average-cpa-mobile-apps.md) убедитесь, что в трекере включена передача данных об установках приложения.

Настройки интеграции подробно описаны в [документации AppsFlyer](https://support.appsflyer.com/hc/en-us/articles/207033816-Integrated-partners-setup-guide#partner-setup).

2. В поисковой строке введите «Yandex Direct».

- **Настройте нужные параметры**

1. Нажмите кнопку **Set up integration**.
2. Активируйте опцию **Activate partner**.

Совет

Для iOS-приложений настройку **Advanced Privacy** оставьте выключенной. Это не повлияет на сбор данных и поможет улучшить обучение стратегий.

1. Скопируйте **Yandex App ID** из библиотеки мобильных приложений. Где найти [App ID](../products-mobile-apps-ads/tracking-systems.md#add-aps).
2. Вставьте скопированный **App ID** в поле **Yandex App ID**.
3. Включите опцию **In-app events postback** для передачи событий.
4. Выберите события, которые вы хотите передавать в Директ. Их можно будет просматривать в Мастере Отчетов и использовать для оптимизации ставок.

Для каждого передаваемого события укажите название, под которым оно будет показываться в Директе.

Выберите способ отправки событий **All media sources, including organic**.

Выберите **Values & revenue** для передачи выручки. Ее можно будет увидеть в Мастере Отчетов.

Нажмите **Add Event** и добавьте **af\_app\_opened**, чтобы включить передачу сессий.

- **Создайте трекинговую ссылку в разделе на вкладке **Attribution Link**.**

  В трекинговых ссылках можно использовать [динамические параметры Директа](../statistics/url-tags.md#dynamic). Если вы планируете использовать атрибуцию на показ в своих кампаниях, убедитесь что она включена в настройках трекера: **Конфигурация** → **Интеграции** → **Атрибуция на показ**.

  Подробнее создание трекинговой ссылки описано в [помощи AppsFlyer](https://support.appsflyer.com/hc/en-us/articles/207033816-Integrated-Partner-Configuration). Если у вас возникли вопросы по работе ссылки, обратитесь в [службу поддержки AppsFlyer](https://support.appsflyer.com/hc/en-us/requests/new).

  Важно

  Убедитесь, что используются обязательные параметры:

  - `pid=yandexdirect_int`;
  - `clickid={logid}`;
  - `advertising_id={google_aid}` — в кампаниях для Android;
  - `idfa={ios_ifa}` — в кампаниях для iOS;
  - `is_retargeting=true` — для ретаргетинговых кампаний.

  Для корректной работы стратегии [Оптимизация конверсий](../strategies/average-cpa-mobile-apps.md) убедитесь, что в трекере включена передача данных об установках приложения.

  Рекомендуем включить передачу статуса органичности данных в трекере — количество передаваемых данных увеличится, а эффективность кампаний вырастет.

  Для передачи статуса перейдите в раздел **Active integrations**, выберите **Yandex Direct** и в блоке **General settings** включите опцию **Share organic/non-organic indication**.

Настройки интеграции подробно описаны в [документации Adjust](https://help.adjust.com/en/integrated-partners/yandex).

12. Чтобы завершить настройку интеграции, нажмите **Create link**.

Настройки интеграции подробно описаны в [документации Branch](https://help.branch.io/using-branch/docs/universal-ad-partner-integration-guide).

- **Выберите партнера Yandex.Direct**

  1. В интерфейсе Branch перейдите на страницу [Ads](https://dashboard.branch.io/ads).
  2. Выберите **Partner Management** → **Yandex.Direct**.
  3. Сохраните изменения.
- **Настройте нужные параметры**

  1. Откройте вкладку **POSTBACK CONFIG**.
  2. Включите отправку событий **INSTALL**.
  3. Включите отправку событий InApp. Для каждого укажите имя события в Директе. Для этого в поле **POSTBACK URL** замените параметр `&event=${(name)!}` на `&event=EVENT_NAME`, где `EVENT_NAME` — одно из значений:

     - `ACHIEVED_LEVEL` — достижение цели (Completed goal);
     - `ADDED_TO_CART` — добавление в корзину (Added to cart);
     - `ADDED_PAYMENT_INFO` — добавление информации об оплате (Added payment info);
     - `ADDED_TO_WISHLIST` — добавление в список желаний (Added to wish list);
     - `APP_LAUNCHED` — запуск приложения (Launched app);
     - `COMPLETED_REGISTRATION` — завершение регистрации (Completed registration);
     - `COMPLETED_TUTORIAL` — завершение обучения (Completed tutorial);
     - `EVENT_1` — пользовательское событие 1 (Custom event 1);
     - `EVENT_2` — пользовательское событие 2 (Custom event 2);
     - `EVENT_3` — пользовательское событие 3 (Custom event 3);
     - `INITIATED_CHECKOUT` — инициирование заказа (Initiated order);
     - `INSTALL_APP` — установка приложения (AppInstall);
     - `PURCHASED` — совершение покупки (Made a purchase);
     - `RATED` — проставление рейтинга (Submitted rating);
     - `SEARCHED` — выполнение поиска (Performed search);
     - `SHARED` — пользователь поделился (User shared);
     - `SPENT_CREDITS` — потрачены кредиты (Spent credits);
     - `SPENT_TIME_IN_APP` — проведение времени в приложении (Time spent in app);
     - `UNLOCKED_ACHIEVEMENT` — достижение разблокировано (Achievement unlocked);
     - `VIEWED_CONTENT` — просмотр контента (Content viewed).
  4. Для верификации приложения для всех событий в поле **POSTBACK URL** добавьте параметр `&ya-app-id=APP-ID`, где `APP-ID` — идентификатор приложения в Директе.
  5. Для всех событий кампаний, которые используют электронную коммерцию, в поле **POSTBACK URL** добавьте параметр `&ecom_feed_counter_id=ECOM-ID`.
  6. Включите отправку всех событий. Для этого поставьте галочку в поле **ALL EVENTS**.

     Это позволит:

     - не таргетироваться на уже установивших приложение пользователей;
     - запускать ретаргетинговые кампании;
     - улучшить эффективность кампаний в целом.
- **Создайте трекинговую ссылку**

  Создайте трекинговую ссылку по [инструкции](https://help.branch.io/using-branch/docs/ad-links).

  Укажите тип ссылки — Ad Link.

  На этапе **Define** в поле **Which Ad Partner is this for?** выберите **Yandex.Direct**.

  Настройте обязательные параметры ссылки:

  - `&click_id={logid}`;
  - `&%243p=a_yandex_direct`.

Настройки интеграции подробно описаны в [документации MyTracker](https://tracker.my.com/docs/tracking/integration/yadirect/about).

Подключение аккаунта:

1. Авторизуйтесь в Директе.
2. Откройте [страницу интеграции](https://tracker.my.com/yadirect/account/) в MyTracker.
3. Выберите аккаунт MyTracker, к которому относятся интересующие вас приложения.
4. Нажмите кнопку **Привязать аккаунт Яндекс Директ**.

   Если у MyTracker еще нет доступа к вашему аккаунту в Яндексе, вы будете перенаправлены на сервер Яндекс OAuth, где нужно нажать кнопку **Разрешить**.

   Через некоторое время на [странице интеграции](https://tracker.my.com/yadirect/account/) появится ваш аккаунт Яндекс Директ со статусом синхронизации. Статистика по нему поступит в течение суток.
5. Чтобы создать трекинговую ссылку, найдите в списке кампаний автоматически созданную кампанию по названию:

   `[Yandex Direct идентификатор кампании::номер связи] Название кампании в Yandex Direct.`
6. На странице **Трекинг-ссылки** добавьте [обычную](https://tracker.my.com/docs/tracking/links/regular_link) или [умную трекинг-ссылку](https://tracker.my.com/docs/tracking/links/smartlink).

## Диплинк

Диплинк (от англ. *deep link*) — это обязательный параметр трекинговой системы, который добавляется в кампанию на ретаргетинг. Он нужен, чтобы пользователь при клике по объявлению переходил в определенный раздел приложения.

Добавьте при создании кампаний на ретаргетинг:

- в [Мастере кампаний](../products-mobile-apps-ads/create.md) в поле **Трекинговая ссылка**;
- в [Единой перфоманс-кампании](../products-mobile-apps-ads/upc/about.md) с целью **Ретаргетинг** в поле **Диплинк** при создании объявления.

Для подготовки диплинка обратитесь к разработчику приложения.

## Отложенный диплинк

Отоложенный диплинк (от англ. *deep link*) — это дополнительный параметр для кампаний с целью **Привлечение новых пользователей**. Он нужен, чтобы пользователь после установки переходил в определенный раздел приложения.

Добавьте:

- в [Мастере кампаний](../products-mobile-apps-ads/create.md) в поле **Трекинговая ссылка**. Для кампаний, запущенных на новую аудиторию;
- в [Единой перфоманс-кампании](../products-mobile-apps-ads/upc/about.md) в поле **Отложенный диплинк** при создании объявления. Доступно только для кампаний с целью **Привлечение новых пользователей**.

Для подготовки диплинка обратитесь к разработчику приложения.
