# Требования к баннеру

- [Технические требования к графическому баннеру](ru/fixed-cpm-campaigns/products-frontpage/requirements#technical-image)
- [Технические требования к HTML5-баннеру](ru/fixed-cpm-campaigns/products-frontpage/requirements#technical)
- [Требования модерации](ru/fixed-cpm-campaigns/products-frontpage/requirements#moderation)

Вы можете загрузить графический или HTML5-баннер. На мобильной версии страницы ya.ru HTML5-баннер не размещается.

Запуск формата «Баннер под поисковой строкой. Кросс» состоится только при наличии в кампании баннеров двух размеров: 1456 × 180 и 640 × 134.

## Технические требования к графическому баннеру

|  |  |  |
| --- | --- | --- |
|  | **Компьютерная версия сайта ya.ru, новая вкладка Браузера** | **Мобильная версия сайта ya.ru** |
| Размер в пикселях | 1456 × 180 | 640 × 134 |
| Максимальный вес файла | 1 МБ | 512 КБ |
| Формат | JPG, PNG, GIF или HTML5 | JPG, PNG или GIF |

## Технические требования к HTML5-баннеру

HTML5-баннер — это ZIP-архив, содержащий один файл HTML и файлы JavaScript, JSON, CSS, JPEG, GIF, PNG, SVG.

|  |  |
| --- | --- |
|  | **Компьютерная версия сайта ya.ru, новая вкладка Браузера** |
| Размер в пикселях | 1456 × 180 |
| Максимальный вес архива | 1 МБ |

Архив и файлы в баннере должны соответствовать требованиям:

1. В названиях файлов и каталогов можно использовать только буквы латинского алфавита, цифры и символы `-._~`.
2. Все ссылки в HTML5-баннере должны быть относительными и вести на другие файлы из архива. При отработке рекламных материалов не должно быть внешних сетевых обращений. При необходимости все вспомогательные JS-библиотеки должны предоставляться вместе с рекламными материалами.
3. В архиве может быть не более 20 файлов. Включите исходный код из файлов JavaScript, CSS, SVG в HTML-файл, а все растровые изображения объедините в один файл (атлас спрайтов).
4. Максимальный вес файла `index.html` в архиве должен быть меньше 150 КБ.
5. В архиве будет игнорироваться содержимое папки \_\_MACOSX и файлы .DS\_Store, Thumbs.db. Это вспомогательные файлы операционных систем, которые не нужны для работы HTML5-баннера.
6. По клику на баннер будет открываться страница, указанная в поле **Ссылка на сайт**, мобильное приложение (если оно установлено) или страница приложения в магазине (если приложение не установлено). Переход на эту страницу должен генерироваться в баннере вызовом метода `yandexHTML5BannerApi.getClickURLNum(1)`.

   ```
   <a id="click_area" href="#" target="_blank">...</a>
   // после элемента "<a href ...></a>"
   // присваиваем для него значение атрибута href так:
   <script>
   document.getElementById("click_area").href = yandexHTML5BannerApi.getClickURLNum(1);
   </script>
   ```
7. В атрибуте `content` тегa `meta` должен быть указан размер HTML5-баннера, как в примере ниже:

   ```
   <meta name="ad.size" content="width=1456,height=180">
   ```
8. Разрешаются внешние вызовы [JavaScript-библиотек, размещенных на серверах Яндекса](https://tech.yandex.ru/jslibs/). Если в HTML5-баннере используется другая библиотека, включите ее исходный код в архив.
9. Все обработчики событий должны быть явно указаны внутри тега `script`, например:

   ```
   <script>
   document.onload = init;
   </script>
   ```
10. В ссылке на сайт, содержащейся в HTML5-баннере, должно быть не более 1024 знаков в кодировке UTF-8.
11. В баннере медийной кампании не должны использоваться инструменты сбора статистики (пиксели, счетчики и прочие).

    При создании кампании вы сможете указать пиксель Яндекс Аудиторий или счетчик ADFOX (подробнее в разделе [Как запустить кампанию Видеобаннер на Поиске](ru/fixed-cpm-campaigns/products-frontpage/create)), а также использовать [UTM-метки](ru/statistics/url-tags) в ссылке на сайт.
12. В баннер можно добавить управляющую кнопку **Посмотреть еще раз**, нажатие на которую приведет к повторному запуску сценария.
13. Рекламируемый сайт должен открываться в новом окне и только после нажатия левой кнопкой мыши по HTML5-баннеру.
14. HTML5-баннер должен работать корректно и одинаково для всех пользователей на всех платформах в течение рекламной кампании в следующих браузерах: Яндекс Браузер 14 и выше, Google Chrome 14 и выше, Mozilla Firefox 14 и выше, Opera 14 и выше, Internet Explorer 9 и выше, Microsoft Edge.
15. В HTML5-баннере нельзя использовать видео- и аудиоролики, скрипты с раздражающим или вредоносным поведением (всплывающие окна, подмена адреса страницы и т. д.), расширения и плагины браузера (Adobe Flash и т. д.). Исходный код баннера не должен быть обфусцирован, то есть иметь вид, затрудняющий анализ.
16. HTML5-баннер должен соответствовать требованиям к загрузке процессора.

    Требования к загрузке процессора

    Для тестирования уровня загрузки процессора используется компьютер с Intel Core 2 Duo P9600 2,53 ГГц, 2 ГБ оперативной памяти, 64–800 МБ видеопамяти и установленной 64-разрядной версией Windows 7 Профессиональная. На нем загружается одна страница, которая содержит только тестируемый баннер. Он не должен отнимать более 50% ресурсов процессора в течение определенного времени (доли секунды или секунды) или более 70% ресурсов процессора в любой момент времени. Загрузка процессора оценивается с помощью Диспетчера задач.

## Требования модерации

##### **Тексты, сайт и отображение баннеров**

Применяются [стандартные правила](ru/moderation/ad-rules#image-ads) к оформлению, сайту и отображению баннеров. Убедитесь, что добавлены все необходимые [предупреждения и возрастные ограничения](ru/moderation/age-alerts#image-ads-and-video).

Кроме этого, у баннера не должно быть рамки.

Рекомендации к [оформлению баннеров](ru/products-cpm-campaign-frontpage/full-version).

##### **Запрещенные тематики**

Для размещения не принимаются баннеры следующих тематик:

- политика;
- магия и эзотерика;
- религиозные товары и услуги;
- услуги коллекторов;
- ритуальные товары и услуги;
- сайты знакомств или брачных агентств;
- социальная реклама, в том числе благотворительные организации;
- услуги детективов и частных охранных предприятий;
- товары и услуги для взрослых (например, секс-игрушки, эротические материалы и т. п.);
- аксессуары для оружия;
- конструктивно сходные с оружием изделия и аксессуары для них;
- военная тематика (перевозка военных, юридическая помощь в получении военного билета и т. п.);
- услуги по оптимизации сайтов и SEO;
- услуги по дезинфекции и дезинсекции;
- некоторые финансовые услуги и обучающие курсы по ним (микрофинансирование, предложение займов от частных лиц, сделки с бинарными опционами, деятельность форекс-дилеров, сделки с форекс-дилерами и т. п.);
- иные тематики, не соответствующие редакционной политике Яндекса.

Этот список не является полным и может быть изменен Яндексом в любой момент.
