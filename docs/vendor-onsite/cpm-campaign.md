# Медийная реклама в интернет-магазинах

- [Шаг 1. Настройте параметры кампании](ru/vendor-onsite/cpm-campaign#shag-1-nastrojte-parametry-kampanii)
- [Шаг 2. Настройте целевую аудиторию](ru/vendor-onsite/cpm-campaign#shag-2-nastrojte-celevuyu-auditoriyu)
- [Шаг 3. Настройте объявление](ru/vendor-onsite/cpm-campaign#shag-3-nastrojte-obuyavlenie)
  - [Технические требования к баннерам](ru/vendor-onsite/cpm-campaign#tehnicheskie-trebovaniya-k-banneram)

Медийная реклама в интернет-магазинах — это тип кампании в Директе для вендоров, который позволяет показывать баннерную рекламу в формате click-in на сайтах и в приложениях интернет-магазинов с оплатой за тысячу показов (CPM).

Объявления в формате click-in переводят пользователя в бренд-раздел, каталог или на карточку товара внутри магазина. Реклама отображается на всех этапах пути покупателя и помогает формировать отношение к бренду.

## Шаг 1. Настройте параметры кампании

1. В меню слева нажмите **Кампания** или на странице **Кампании** нажмите **Добавить кампанию** и выберите **Медийная кампания click-in**.
2. Выберите стратегию показа объявлений, бюджет, период кампании и цену за тысячу показов.
3. Задайте сроки проведения кампании, [расписание показов](ru/efficiency/timetargeting) и [ограничение на пользователя](ru/campaigns/campaign-settings#frequency-capping).

## Шаг 2. Настройте целевую аудиторию

Для каждого магазина создайте отдельную группу объявлений и настройте ее параметры:

1. На основе фидов ритейлеров и данных из профиля вендора система предложит список доступных вам магазинов (сайтов и приложений ритейлеров) сети [Ритейл Медиа](https://retailmedia.yandex.ru/promo). В блоке **Магазин** выберите один из них.
2. Настройте [географию показов](ru/efficiency/geotargeting).

   Если регион показа не Россия, выберите язык объявлений — так объявления будут показываться только тем потенциальным клиентам, которые понимают выбранный язык или посещают интернет-ресурсы на этом языке.
3. Настройте [социально-демографический профиль](ru/products-cpm-campaign/display-ads-settings#demography) аудитории.

## Шаг 3. Настройте объявление

Для каждого из выбранных магазинов создайте одно или несколько объявлений:

1. Укажите в объявлении [ссылку](ru/efficiency/display-link) на страницу внутри магазина. Ссылка может вести на карточку товара, подборку предложений (сезонных или со скидками) или на все товары бренда.
2. Загрузите креативы с компьютера или выберите из ранее загруженных. Креативы должны соответствовать [требованиям модерации](ru/moderation/ad-rules#image-ads).

### Технические требования к баннерам

1. Максимальный объем файла: 512 КБ.
2. Формат файла:

   - ZIP (для HTML5-баннеров);
   - JPG, PNG или GIF (для графических креативов).
3. При загрузке креатива система покажет, какие размеры рекламных блоков поддерживает конкретный сайт ритейлера. Размер HTML5-баннера в пикселях должен совпадать с размером рекламного блока, а размер графического креатива может быть больше в 2–3 раза, но с сохранением соотношения сторон.

4. При подготовке креатива учитывайте, что верхняя область (сейфзона) может перекрываться элементами интерфейса, поэтому на ней не следует размещать текст, логотип и другие элементы.

**Требования к HTML5-баннерам**

HTML5-баннер — это ZIP-архив, содержащий один файл HTML и файлы JavaScript, JSON, CSS, JPEG, GIF, PNG, SVG.

Архив и файлы в HTML5-баннере должны соответствовать требованиям:

1. В названиях файлов и каталогов можно использовать только буквы латинского алфавита, цифры и символы `-._~`.
2. Все ссылки в HTML5-баннере должны быть относительными и вести на другие файлы из архива.
3. В архиве может быть не более 20 файлов. Включите исходный код из файлов JavaScript, CSS, SVG в HTML-файл, а все растровые изображения объедините в один файл (атлас спрайтов).
4. Максимальный вес файла `index.html` в архиве должен быть меньше 150 КБ. Максимальный вес самого архива — 512 КБ.
5. В архиве будет игнорироваться содержимое папки \_\_MACOSX и файлы .DS\_Store, Thumbs.db. Это вспомогательные файлы операционных систем, которые не нужны для работы HTML5-баннера.
6. По клику на баннер будет открываться страница, указанная в поле **Ссылка на сайт**, мобильное приложение (если оно установлено) или страница приложения в магазине (если приложение не установлено). Переход на эту страницу должен генерироваться в баннере вызовом метода `yandexHTML5BannerApi.getClickURLNum(1)`.

   ```
   <a id="click_area" href="#" target="_blank">...</a>
   // после элемента "<a href ...></a>"
   // присваиваем для него значение атрибута href так:
   <script>
      document.getElementById("click_area").href = yandexHTML5BannerApi.getClickURLNum(1);
   </script>
   ```
7. Разрешаются внешние вызовы [JavaScript-библиотек, размещенных на серверах Яндекса](https://tech.yandex.ru/jslibs/). Если в HTML5-баннере используется другая библиотека, включите ее исходный код в архив.

HTML5-баннер должен соответствовать правилам Директа:

1. Рекламируемый сайт должен открываться в новом окне и только после нажатия левой кнопкой мыши по HTML5-баннеру.
2. HTML5-баннер должен работать корректно и одинаково для всех пользователей на всех платформах в течение рекламной кампании в следующих браузерах: Яндекс Браузер 14 и выше, Google Chrome 14 и выше, Mozilla Firefox 14 и выше, Opera 14 и выше, Internet Explorer 9 и выше, Microsoft Edge.
3. В HTML5-баннере нельзя использовать видео- и аудиоролики, скрипты с раздражающим или вредоносным поведением (всплывающие окна, подмена адреса страницы и т. д.), расширения и плагины браузера (Adobe Flash и т. д.). Исходный код баннера не должен быть обфусцирован, то есть иметь вид, затрудняющий анализ.
4. HTML5-баннер должен соответствовать требованиям к загрузке процессора.

   Требования к загрузке процессора

   Для тестирования уровня загрузки процессора используется компьютер с Intel Core 2 Duo P9600 2,53 ГГц, 2 ГБ оперативной памяти, 64–800 МБ видеопамяти и установленной 64-разрядной версией Windows 7 Профессиональная. На нем загружается одна страница, которая содержит только тестируемый баннер. Он не должен отнимать более 50% ресурсов процессора в течение определенного времени (доли секунды или секунды) или более 70% ресурсов процессора в любой момент времени. Загрузка процессора оценивается с помощью Диспетчера задач.
5. Нельзя использовать резко перемещающиеся и мелькающие графические элементы, мигающий фон. Время переключения между кадрами должно быть достаточным для прочтения всего показанного текста.

После настройки всех параметров кампании, групп и объявлений, нажмите **Запустить кампанию**.
