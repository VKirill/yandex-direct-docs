# Офлайн-конверсии

- [Зачем передавать офлайн-конверсии](../strategies/offline-conversions.md#for-what)
- [Подготовительные работы для передачи данных](../strategies/offline-conversions.md#preparation)
- [Качество атрибуции конверсии к визиту](../strategies/offline-conversions.md#crm-connection)
- [Способы передачи конверсий](../strategies/offline-conversions.md#transfer-methods)

Офлайн-конверсии — это конверсии, которые хранятся в вашей базе CRM, но не отслеживаются на сайте. Например:

- пользователь посмотрел товар на сайте, но купил его офлайн;
- пользователь выбрал товар на сайте, но оплатил покупку в магазине или совершил звонок, который привел к оплате.

## Зачем передавать офлайн-конверсии

Дополните статистику конверсиями, которые происходят вне сайта. Это позволит проанализировать размещение по всем данным и оптимизировать все типы конверсий.

- Принимайте решение об инвестировании на основе полных данных, из которых вы узнаете:

  - какие кампании приносят офлайн-конверсии;
  - финальную стоимость по всем конверсиям в ваших кампаниях.
- Используйте офлайн-конверсии для оптимизации кампаний.

  Алгоритмы Директа получат больше информации о пользователях, совершающих конверсии вне сайта. Это повысит эффективность работы стратегии: чем больше данных получает стратегия — тем лучше понимает портрет целевого пользователя, которому нужно показать объявление.

## Подготовительные работы для передачи данных

- Установите и настройте [счетчик](https://yandex.ru/support/metrica/general/creating-counter.html) Метрики — сопоставление офлайн-конверсий с визитами на сайт происходит благодаря технологиям Метрики.
- У вас должна быть CRM-система, в которой фиксируется информация о пользователях и как онлайн, так и офлайн-заказах.
- Для отправки данных обязателен хотя бы один из идентификаторов пользователя:

  - ClientID;
  - телефон;
  - почта.
- Настройте сбор и передачу ClientID по онлайн-заказам. Это позволит увеличить долю целевых визитов (в процентах) на сайт.
  Вы сможете отслеживать:

  - статистику по заказам из CRM-системы c различными типами статуса заказа;
  - статистику по целям **CRM: Заказ создан** и **CRM: Заказ оплачен**.

  Также появится возможность передавать те конверсии, у которых не зафиксирован ClientID.

## Качество атрибуции конверсии к визиту

Чем больше идентификаторов известно о каждой конверсии, тем больше шанс успешной атрибуции конверсии к конкретному визиту. Передавайте специальный идентификатор [ClientID](https://yandex.ru/support/metrica/data/offline-params.html#offline-params__ids) по тем конверсиям, где их можно собрать.

Качество атрибуции зависит от данных, передаваемых по той или иной конверсии.

|  |  |  |
| --- | --- | --- |
| **Передаете ClientID?** | **Передаете телефон или email?** | **Качество атрибуции** |
| Да | Да | Максимальное |
| Да | Нет | Хорошее |
| Нет | Да | Среднее (1/2 от максимальной атрибуции) |

Чтобы атрибуция работала максимально точно, запоминайте ClientID для всех онлайн-заявок и заказов. Используйте JavaScript метод [getClientID](https://yandex.ru/support/metrica/objects/get-client-id.html) и указывайте ClientID при передаче данных.

## Способы передачи конверсий

Для передачи офлайн-конверсий в Метрику можно использовать:

- [Центр конверсий](../statistics/conversions.md) — загрузите файл с конверсиями через FTP или SFTP сервер, по ссылке на HTTP или HTTPS или через Google Sheets и добавьте ссылку на источник в Директе.
- Интеграцию CRM-системы с Метрикой через коннектор (со списком доступных коннекторов можно ознакомиться в [интерфейсе Метрики](https://metrika.yandex.ru/marketplace?id=44147844&filter=61389b8d25c17900556b3077)).
- [API](https://yandex.ru/dev/metrika/doc/api2/practice/crm/contacts.html) — отправка данных доступна в двух форматах: Упрощенная передача данных и Передача подробных данных.
