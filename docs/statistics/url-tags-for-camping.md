# Как добавить параметры URL в кампании

- [Как добавить единые параметры](../statistics/url-tags-for-camping.md#how-add)
- [Какие выстраиваются приоритеты по параметрам](../statistics/url-tags-for-camping.md#campaign-tags)

## Как добавить единые параметры

В блоках **Параметры URL** укажите общие метки, в том числе UTM, и динамические параметры:

- при создании или редактировании **кампании** — метки и параметры автоматически добавятся к каждой ссылке всех объявлений **кампании** из всех групп, то есть к ссылкам, быстрым ссылкам и кнопке.
- при создании или редактировании **группы** — метки и параметры автоматически добавятся к каждой ссылке всех объявлений **группы**, то есть к ссылкам, быстрым ссылкам и кнопке.
- при создании или редактировании **объявления** — параметры и метки буду автоматически добавлены к ссылке на сайт, быстрым ссылкам и кнопке.

Блоки **Параметры URL** доступны в [Единой перфоманс-кампании](../unified-performance-campaign/create-campaign.md).

После обновления параметров URL объявления отправляются на повторную модерацию.

## Какие выстраиваются приоритеты по параметрам


1. Параметры из блоков **Параметры URL** имеют приоритет над параметрами, указанными в поле **Ссылка на сайт** в объявлении. Если в ссылке и в блоке будут пересекающиеся значения одного и того же параметра, значение возьмется из блока **Параметры URL**.

   > Например, если на уровне кампании вы задали параметры URL:
   >
   > ```
   > utm_source=yandex&utm_medium=cpc
   > ```
   >
   > В объявлениях в ссылке указали:
   >
   > ```
   > re-store.ru/?utm_source=ya&utm_medium=cpc&utm_name=brand
   > ```
   >
   > Итоговая ссылка будет содержать:
   >
   > ```
   > re-store.ru/?utm_source=yandex&utm_medium=cpc&utm_name=brand
   > ```
   >
   > где:
   >
   > `utm_source=yandex` — значение подтянется из блока **Параметры URL**, заданное на уровне кампании, вместо `utm_source=ya`, указанного в ссылке объявления.
   >
   > `utm_medium=cpc` — значение параметра останется неизменным, так как оно пересекается в ссылке и в блоке **Параметры URL**.
   >
   > `utm_name=brand` — значение останется неизменным, так как в блоке **Параметры URL** такого параметра нет.
2. Если вы задаете параметры в блоках **Параметры URL** на кампанию, в группах и в объявлениях, то к ссылкам в объявлениях они будут добавляться по следующим правилам:

   1. Параметры для объявления имеют приоритет над параметрами для группы и на кампанию.
   2. У параметров для группы приоритет над параметрами на кампанию.

   То есть к ссылке в объявлении и быстрым ссылкам в первую очередь будут добавлены метки из блока **Параметры URL** на уровне объявления. Если они не заданы, то метки из блока на уровне группы. Если не заданы метки на уровнях объявления или группы, то метки на кампанию.

   Если в ссылке объявления уже указаны метки, то метки из блока **Параметры URL** будут добавлены к ним. Но если в ссылке и в блоке будут пересекающиеся значения одного и того же параметра, значение возьмется из блока **Параметры URL**.

   #### Пример 1

   > На уровне кампаний вы задали параметры URL:
   >
   > ```
   > utm_medium=cpc&utm_source=yandex
   > ```
   >
   > Параметры на уровне группы:
   >
   > ```
   > utm_medium=cpc&utm_source=yandex&utm_term={ad_id}
   > ```
   >
   > Ссылка в объявлении:
   >
   > ```
   > re-store.ru/?utm_term={keyword}&utm_content=HappyNY#ancor
   > ```
   >
   > Итоговая ссылка:
   >
   > ```
   > re-store.ru/?utm_medium=cpc&utm_source=yandex&utm_term={ad_id}&utm_content=HappyNY#ancor
   > ```
   >
   > где:
   >
   > `utm_medium=cpc` — значение подтянется из блока **Параметры URL**, заданное на уровне группы.
   >
   > `utm_source=yandex` — значение подтянется из блока **Параметры URL**, заданное на уровне группы.
   >
   > `utm_term={ad_id}` — значение подтянется из блока **Параметры URL**, заданное на уровне группы, вместо `utm_term={keyword}`, указанного в ссылке объявления.
   >
   > `utm_content=HappyNY#ancor` — значение останется неизменным, так как в блоке **Параметры URL** на уровне группы и кампании такого параметра нет.

   #### Пример 2

   > На уровне кампании вы задали параметры :
   >
   > ```
   > utm_medium=cpc&utm_source=yandex
   > ```
   >
   > На уровне группы параметры не заданы.
   >
   > Ссылка в объявлении:
   >
   > ```
   > re-store.ru/?utm_term={keyword}&utm_content=HappyNY#ancor
   > ```
   >
   > Итоговая ссылка:
   >
   > ```
   > re-store.ru/?utm_medium=cpc&utm_source=yandex&utm_term={keyword}&utm_content=HappyNY#ancor
   > ```
   >
   > где:
   >
   > `utm_medium=cpc` — значение подтянется из блока **Параметры URL**, заданное на уровне кампании.
   >
   > `utm_source=yandex` — значение подтянется из блока **Параметры URL**, заданное на уровне кампании.
   >
   > `utm_term={keyword}` — значение останется неизменным, так как в блоке **Параметры URL** на уровне кампании такого параметра нет.
   >
   > `utm_content=HappyNY#ancor` — значение останется неизменным, так как в блоке **Параметры URL** на уровне кампании такого параметра нет.
