## Специальные параметры {param1} и {param2}

Динамические параметры `{param1}` и `{param2}` — параметры широкого назначения. Во-первых, их можно использовать как дополнительные параметры для сбора статистики, список передаваемых значений для которых вы задаете вручную. Во-вторых, с помощью `{param1}` и `{param2}` можно переадресовывать на разные страницы пользователей, пришедших по одному объявлению, но по разным ключевым фразам.

Если вы используете специальные параметры URL, **обязательно проверьте**, какая ссылка сформируется для [автотаргетинга](ru/impression-criteria/autotargeting). Найдите в списке служебные фразы `---autotargeting`. Посмотрите, что указано для них в `{param1}` и `{param2}`. Вы можете заполнить их какими-то подходящими значениями или оставить пустыми, если ссылка, указанная для автотаргетинга, работает.

Задать значения `{param1}` и `{param2}` можно только в [файлах формата XLS/XLSX](ru/alternative-interfaces/xls-interface), в [Директ Коммандере](ru/alternative-interfaces/direct-commander) или через [API](https://tech.yandex.ru/direct/doc/examples-v5/curl-ads-keywords-modify-docpage/).

> Например, задайте значения в таблице Excel:
>
> |  |  |  |
> | --- | --- | --- |
> | **Фраза (с минус-словами)** | **Ссылка** | **Параметр 1** |
> | купить велосипед | `http://site.ru/catalog/{param1}` | bikes.html |
> | купить самокат | `http://site.ru/catalog/{param1}` | scooters.html |
> | `---autotargeting` | Вариант 1:  `http://site.ru/catalog/all/` |  |
> | Вариант 2:  `http://site.ru/catalog/{param1}` |  |
> | Вариант 3:  `http://site.ru/catalog/{param1}` | bikes.html |
>
> Объявление, которое будет показано по ключевой фразе *купить велосипед*, будет вести на страницу `http://www.site.ru/catalog/bikes.html`.
>
> Объявление, которое будет показано по ключевой фразе *купить самокат*, будет вести на страницу `http://www.site.ru/catalog/scooters.html`.
>
> Объявление, которое будет показано с помощью автотаргетинга:
>
> - будет вести на страницу `http://www.site.ru/catalog/all/`, при условии, что ссылка рабочая и открывает весь каталог сайта, например (вариант 1);
> - без указания значения `{param1}` будет вести на страницу `http://www.site.ru/catalog/`, при условии, что ссылка рабочая и открывает весь каталог сайта, например (вариант 2);
> - с указанием подходящего значения `{param1}` будет вести на страницу `http://www.site.ru/catalog/bikes.html` (вариант 3).

Внимание

- Ссылка, указанная в загруженном файле, проверяется без параметров `{param1}` и `{param2}`. Например, если указана ссылка `http://site.ru/{param1}`, то система проверит страницу по ссылке `http://site.ru/`. Убедитесь, что эта страница доступна и корректно отображается во всех браузерах.
- Специальные символы в значениях параметров `{param1}` и `{param2}` автоматически кодируются в UTF-8. Например, если в качестве параметра `{param1}` использовать «flowers/lilac.html», символ «/» будет преобразован в последовательность «%2F», и объявление будет вести на страницу `http://www.site.ru/flowers%2Flilac.html`.
- Если вы используете в загруженном файле ссылку вида `http://site.ru/{param1}/all`, для автотаргетинга кроме ссылки обязательно указывайте значение `{param1}`. Если в файле указать только сайт без параметров, переход по такой ссылке приведет на несуществующую страницу вида `http://www.site.ru//all`.

## Использование параметров в сочетании с шаблонами

Параметры можно использовать совместно с [шаблонами](ru/features/ad-templates):

```
http://www.site.ru/?key=##&ad_id={ad_id}
```

Убедитесь, что страницы, на которые ведут ссылки с шаблонами, доступны.

## Применение специальных параметров для перехода из одного объявления на разные страницы сайта

Чтобы при клике на объявление пользователь переходил на разные страницы в зависимости от ключевой фразы, вы можете использовать [шаблон](ru/features/ad-templates) вместе со [специальными параметрами](ru/statistics/url-tags#param1).

Это можно сделать с помощью [файлов формата XLS/XLSX](ru/alternative-interfaces/xls-interface), в [Директ Коммандере](ru/alternative-interfaces/direct-commander) или через [API](https://tech.yandex.ru/direct/doc/examples-v5/curl-ads-keywords-modify-docpage/). Если у вас несколько групп однотипных объявлений, объедините такие объявления в одну группу. Для этого измените номер группы у всех объявлений, которые вы хотите объединить. Для каждого объявления задайте уникальную ключевую фразу, в заголовки и текст добавьте шаблон. В ссылку для всех объявлений добавьте параметр `{param1}`. Его значения задайте в столбце «Параметр 1».

Если на страницах перехода нужно указать сразу несколько переменных, то используйте `{param1}` и `{param2}` вместе. Например, информация на вашем сайте сгруппирована по производителям и для каждого артикула создана отдельная страница вида `http://site.ru/trudi/giraffe.html`, `http://site.ru/aurora/elephant.html`.

Укажите ссылки в формате `http://site.ru/{param1}/{param2}`. Для каждого объявления задайте производителя в столбце «Параметр 1» и артикул в «Параметр 2».

С шаблоном и специальными параметрами

Без шаблона и специальных параметров

|  | Группа объявлений 1 | Группа объявлений 1 |
| --- | --- | --- |
| Ключевая фраза (с минус-словами) | жираф | слон |
| Заголовок | Милый #зоопарк# для души | Милый #зоопарк# для души |
| Текст | Плюшевый #зоопарк# на радость взрослым и детям | Плюшевый #зоопарк# на радость взрослым и детям |
| Ссылка | `http://site.ru/{param1}/{param2}` | `http://site.ru/{param1}/{param2}` |
| Параметр 1 | trudi | aurora |
| Параметр 2 | giraffe.html | elephant.html |

|  | Группа объявлений 1 | Группа объявлений 2 |
| --- | --- | --- |
| Ключевая фраза (с минус-словами) | жираф | слон |
| Заголовок | Милый жираф для души! | Милый слон для души! |
| Текст | Плюшевый жираф на радость взрослым и детям | Плюшевый слон на радость взрослым и детям |
| Ссылка | `http://site.ru/trudi/giraffe.html` | `http://site.ru/aurora/elephant.html` |
| Параметр 1 |  |  |
| Параметр 2 |  |  |

Объявление с текстом *Плюшевый жираф на радость взрослым и детям* будет вести на страницу `http://site.ru/trudi/giraffe.html`, а с заголовком *Милый слон для души* — на страницу `http://site.ru/aurora/elephant.html`.

В результате ключевые фразы в одной группе объявлении наберут достаточное количество показов и ваша группа объявлений не будет получать статус «Для прогноза трафика не хватает статистики».
