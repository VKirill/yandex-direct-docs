«Доход» и «Ценность конверсий» — метрики, которые помогают отслеживать результаты продвижения в Директе с учетом бизнес-показателей. Эти метрики позволяют увидеть, как продвижение влияет на прибыль бизнеса.

## Метрика «Доход»

Метрика «Доход» рассчитывается одинаково во всех отчетах и не зависит от настроек кампаний. При этом учитываются только цели, данные по которым передаются из Метрики, мобильных трекеров или офлайн-конверсии. Например, это могут быть такие цели, как «Ecommerce: покупка», «JavaScript-событие» и другие цели, по которым данные поступают в Директ.

Подробнее, [как передавать доход по цели](https://yandex.ru/support/metrica/general/price-goal.html) в статье Яндекс Метрики.

Примечание

С 16 апреля 2025 года доход в статистике Директа рассчитывается только по целям, которые вы выбрали и по которым передается доход. Показатель автоматически пересчитан за предыдущие периоды.

Если вы вручную измените список выбранных целей, доход по ним за предыдущие периоды также будет рассчитан с учетом нового подхода.

### Как выбрать цели для расчета дохода

В боковом меню перейдите в раздел **Конверсии** → **Настройка метрики «Доход»** и укажите цели, по которым будет рассчитываться доход.

В блоке выбора целей по умолчанию включена опция **Показать только цели, по которым передается Доход**.

На странице по умолчанию выбраны подходящие цели, по которым передается доход: «Ecommerce: покупка», оплата в приложении или оплаченный заказ из CRM. При необходимости скорректируйте список целей. Для каждой цели есть подсказка, которая поможет понять, передавался ли по ней доход.

Нажмите **Сохранить**. Если не сохранить список вручную, Директ может адаптировать его для расчета дохода. Например, если вы настроили новую цель, по которой передается доход, Директ обновит список целей.

### Как рассчитывается доход по выбранным целям

По целям, которые вы выбрали, рассчитывается доход, а также производные от него метрики: ДРР, ROI и прибыль.

Цель, по которой не передается доход, не будет включена в расчет, пока данные не начнут поступать в Директ.

Если вы выбрали несколько целей, доход будет рассчитан как сумма по всем целям. Это могут быть цели, данные по которым передаются из Метрики, мобильных трекеров или офлайн-конверсии.

## Метрика «Ценность конверсий»

Метрика «Ценность конверсий» отражает вашу оценку: какую экономическую выгоду принесет достижение целей. Ценность можно задать вручную в настройках стратегии или передать с помощью Метрики, мобильных трекеров, офлайн-конверсий. Также доступны производные от нее метрики: ДРР, ROI и прибыль по ценности.

### Когда полезна информация о ценности конверсий

- Кампания оптимизируется по ДРР на несколько целей. При этом по одной части целей данные о доходе передаются из Метрики, а по другой — задаются вручную в настройках стратегии.

  Например, в кампании заданы цели «Ecommerce: покупка» с динамической ценностью и «Ecommerce: добавление в корзину» с заданной вручную ценностью.
- Необходимо проанализировать работу кампании по ценности цели, которую задали вручную, а не по передаваемым из Метрики данным.

  Например, по цели «Добавление в корзину» из Метрики отправляется доход, который равен суммарной стоимости товаров в корзине. В таком случае бизнес может указать ценность цели «Добавление в корзину» вручную в настройках стратегии.
- Данные о доходе по цели не передаются в Директ, а задаются вручную в стратегии кампании через ценность цели.

  Например, в кампании задана цель «Отправка формы», по которой не передается доход из Метрики. В этом случае бизнес может задать ценность цели самостоятельно — в настройках стратегии.
- Нужно анализировать данные о доходе по нескольким важным для бизнеса целям отдельно.

  Например, если у интернет-магазина есть цели «Офлайн-покупка» и «Онлайн-покупка», которые выбраны для расчета метрики «Доход».

### Как рассчитывается ценность конверсий

В первую очередь учитывается ценность, заданная вручную в настройках стратегии кампании. Если ценность достижения цели не задана, то она рассчитывается по данным о доходе, которые передаются из Метрики, мобильных трекеров или офлайн-конверсий.

Чтобы проанализировать ценность конверсий по конкретным целям, выберите их в настройках Мастера отчетов. Если цели не указаны, ценность будет рассчитана по целям, которые были заданы в настройках стратегии кампании.

Пример 1

> В интернет-магазине косметики через электронную коммерцию в Метрике передаются данные о доходе. В метрике «Доход» настроена одна цель — «Ecommerce: покупка».
>
> В кампании настроены две цели: «Ecommerce: покупка» с динамической ценностью и «Ecommerce: добавление в корзину» с заданной вручную ценностью 100 рублей.
>
> За период было совершено 65 покупок на сумму 300 000 рублей, в корзину товары добавили 100 раз.
>
> В новом Мастере отчетов будут следующие данные:
>
> - Доход = 300 000 рублей.
> - Ценность конверсий = 310 000 рублей.
>
> Доход рассчитывается по цели, которая выбрана в настройках метрики «Доход»: «Ecommerce: покупка» — 300 000 рублей.
>
> Ценность конверсии определяется на основании целей, которые были заданы в стратегии кампании:
>
> - «Ecommerce: покупка» с динамической ценностью — 300 000 рублей.
> - «Ecommerce: добавление в корзину» с заданной вручную ценностью 100 рублей.
>
> Ценность конверсий составит: 300 000 рублей + 100 рублей \* 100 добавленных в корзину товаров = 310 000 рублей.

Пример 2

> В кампании настроена стратегия с ограничением ДРР, равным 15%, и настроены две цели для оптимизации: «Ecommerce: покупка» с динамической ценностью и «Ecommerce: добавление в корзину» с заданной вручную ценностью в 200 рублей. Необходимо проанализировать, укладывается ли стратегия в заданное ограничение по ДРР.
>
> Из Метрики был передан доход, равный 9 689 рублей по покупке и 34 059 рублей по корзине. В корзину товары добавили 10 раз. Расход составил 1 409 рублей.
>
> Ценность конверсий составит: 9 689 + 2 000 = 11 689 рублей.
>
> ДРР по ценности составит: 1 409 / 11 689 = 12%.
>
> Фактический ДРР ниже заданного ограничения. Таким образом, кампания укладывается в целевое значение ДРР.
