# Добавление приложения в библиотеку Директа

- [Настройте цели для приложения](../web-app/add-aps.md#app-goals)
- [Доступ к целям для других аккаунтов](../web-app/add-aps.md#access)

Добавьте информацию о мобильном приложении в Директ. Приложения для разных платформ — iOS и Android — добавляются по отдельности.

1. В меню слева нажмите **Библиотека** и выберите **Мобильные приложения**. Нажмите кнопку **Добавить приложение**.
2. Укажите ссылку на приложение в AppStore или Google Play. Директ загрузит из магазина иконку, цену, рейтинг, количество оценок и технические характеристики приложения.

   Если приложение ранее уже было добавлено в Библиотеку вашего аккаунта Директа и вы верифицировали его в трекере (указали App ID), то делать это повторно не нужно.
3. Укажите настройки в зависимости от трекера:

   AppMetrica

   AppsFlyer

   Adjust

   Свяжите приложение в библиотеке Директа с нужным проектом в AppMetrica — в поле **Приложение из AppMetrica** выберите в списке соответствующее название.

   Если в списке нет нужного названия проекта, перейдите в AppMetrica и выдайте доступ к приложению [по инструкции](https://appmetrica.yandex.ru/docs/common/mobile-settings/access.html), указав логин из Директа.

   Поле **Трекинговая ссылка** оставьте пустым.

   В блоке **Цели для использования в стратегии** [выберите события](../web-app/add-aps.md#app-goals), на которые будете оптимизироваться или по которым будет построен сегмент для ретаргетинга.

   Поле **Трекинговая ссылка** оставьте пустым.

   В блоке **Цели для использования в стратегии** [выберите события](../web-app/add-aps.md#app-goals), на которые будете оптимизироваться.

   Сохраните приложение — оно появится в списке. Идентификатор в столбце **App ID** понадобится вам при интеграции с трекером.

   Поле **Трекинговая ссылка** оставьте пустым.

   В блоке **Цели для использования в стратегии** [выберите события](../web-app/add-aps.md#app-goals), на которые будете оптимизироваться.

   Сохраните приложение — оно появится в списке. Идентификатор в столбце **App ID** понадобится вам при интеграции с трекером.

Внимание

Если у вас разветвленная структура аккаунтов в Директе, то добавлять приложение лучше в библиотеку аккаунта, где ведутся кампании типа «Продвижение приложений».

## Настройте цели для приложения

Для каждого из приложений, которые вы хотите использовать в сценарии Web+App, настройте цели:

1. Откройте **Библиотека** → **Мобильные приложения**. Около нужного приложения нажмите **Редактировать**.
2. В блоке **Цели для использования в стратегии** добавьте нужные. Лучше использовать цели, равноценные событиям с сайта, которые вы используете в оптимизации. Например, чтобы в ЕПК начала показываться аудитория пользователей из приложения, настройте три обязательных цели:

   - Просмотр страницы товара;
   - Добавление товара в корзину;
   - Покупка.

   Идентификатор события слева должен соответствовать идентификатору [события в трекере](../web-app/tracker-setting.md#setup). При этом название цели, которое вы будете видеть в интерфейсе Директа, может быть кастомным.

   Если вы настраиваете передачу событий из AppMetrica, в левом столбце выбирайте название события, как оно указано в отчетах самого трекера. Для оптимизации на ECommerce-события выбирайте события, название которых начинается с префикса `ECOMMERCE_`. Воспользуйтесь строкой **Поиск**, чтобы найти конкретные названия событий.

Цели из приложения можно использовать для оптимизации кампании или в сегментах ретаргетинга.

## Доступ к целям для других аккаунтов

Вы можете использовать цели мобильных приложений в кампаниях со сценарием Web+App сразу на нескольких аккаунтах. Для этого выдайте доступ к целям мобильного приложения из того аккаунта, где находится верифицированное в трекере мобильное приложение.

1. Откройте **Библиотека** → **Мобильные приложения** и нажмите **Предоставить доступ**.
2. Во всплывающем окне укажите логины, которым хотите открыть доступ к целям, и нажмите **Предоставить доступ**.
